Installing Polymer

Polymer is distributed in three forms. Firstly, as "tarballs" - or compresses archives. Secondly, as ZIP files, a more Windowzy archive. Finally, as a subversion repository. Only the last is truly Up To Date. With both the repository and the archives, I do try not to relase actual broken code, but this does happen sometimes. In practise, the repository gets more broken versions, but also gets fixed quicker.

Windows

You need:

Python 2.3
Python is a scripting language. Polymer is entirely written in Python.
wxPython 2.5.x
Polymer uses the corss-platform wxPython toolkit to provide the user interface.
[optional] Tortoise SVN
Optionally, you can download and install Tortoise SVN, which will give you easy updates of Polymer, and also allow you to easily make your own changes and contribute them back if you want.

Unzip the zip file, or use Tortoise SVN to fetch from the repository at http://217.155.137.62/svn/projects/infotrope/polymer/. You'll want to launch polymer.pyw, probably by double clicking on it.

Linux

You need pretty much the same as Windows. The biggest difference is that you probably want the commandline svn, and you probably have Python 2.3 installed already. Oh, and the tarball's easier for you. Just execute polymer.pyw.

Mac OS X

You have Python 2.3, most likely. Just install wxPython. I apologise in advance for the bad interface, it's not Mac-like at all. (Buy me a Mac and I'll redo the interface Mac-style.) Execute polymer.pyw, but use pythonw to do it.

Note for SVN users

Sensible usage, especially at this stage, is to use SVN. You'll be able to update Polymer very easily. You may note that what's in the tarballs is actually two projects - Polymer itself and something called the Infotrope Python Library. Pay no attention to the man behind the curtain... But be aware that I do use the library in other places, including for website work and, hopefully, on my mobile telephonic device thingy.
